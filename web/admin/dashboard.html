Domínio e acesso

Link do cliente (provedor): o provedor acessa o portal por um dos endereços abaixo. Envie o link ao cliente.

Ex.: https://191.252.210.31/ka/portal/ — site do provedor: —

Com domínio próprio: preencha Domínio próprio e peça para o provedor apontar o DNS (CNAME ou A) para este servidor. Acesso: https://<domínio>/portal/.

Provedor aponta DNS para este servidor
slug.BASE_DOMAIN/portal/
Instalação nativa (sem Docker)

Portal, site e RADIUS rodam diretamente neste servidor (systemd: multi-portal, freeradius-standalone). PostgreSQL e FreeRADIUS instalados nativamente.

Para reiniciar: sudo systemctl restart multi-portal ou use o botão Reiniciar RADIUS na aba RADIUS deste painel. Via terminal: sudo systemctl restart freeradius-standalone. Logs: journalctl -u multi-portal -f.

Domínio do provedor

Duas formas de acesso: subdomínio (ex: test01.otyisisp.otnsoft.com.br) ou domínio próprio do provedor (ex: portal.empresa.com.br).

Preencha com o domínio que o provedor vai usar. O provedor configura no DNS dele: CNAME ou registro A apontando para o mesmo servidor onde este SaaS está. Depois acessa https://portal.empresa.com.br/portal/ — o sistema identifica o tenant pelo domínio.
Usado quando o provedor ainda não tem domínio próprio. Acesso: https://<subdomínio>.BASE_DOMAIN/portal/
Snippet Nginx — acesso por path

Cole dentro do server { } do Nginx. Coloque location /slug/portal/ antes de location /slug/.
